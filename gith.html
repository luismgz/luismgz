GitHub Tutorial

What is GitHub?

GitHub is a web-based platform for version control and collaboration. It allows developers to store, manage, and track changes to their code.

Key Concepts

  • Repository (Repo): A project's folder containing all files and version history.
  • Commit: A saved change to one or more files in your repository.
  • Branch: A parallel version of a repository, used to work on features without affecting the main code.
  • Pull Request: Proposed changes to a repository that can be reviewed and merged.

Basic GitHub Workflow

  1. Create a repository or clone an existing one.
  2. Create a branch to work on a new feature.
  3. Make changes and commit them to your branch.
  4. Open a pull request to propose your changes.
  5. Discuss and review your code with collaborators.
  6. Merge your pull request once it's approved.
